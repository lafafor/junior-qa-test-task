# Чек-лист тестирования оплаты заказа

**Задача:** Написание тест-кейсов для веб-приложения по оплате заказа банковской картой через внешний платежный шлюз.

---

## 1. Позитивное тестирование / Клиентская валидация

- [ ] **1.1 Номер карты** — 16 символов, пробелы, определение типа (VISA, Mastercard)
- [ ] **1.2 Срок карты** — не прошлый месяц/год
- [ ] **1.3 CVV** — состоит из 3 цифр
- [ ] **1.4 Имя держателя** — без спецсимволов (!"№%;, и т.д.)
- [ ] **1.5 Чек-бокс** — обязательное согласие с активной ссылкой на политику конфиденциальности
- [ ] **1.6 3-D Secure** — перенаправление на страницу банка-эмитента
- [ ] **1.7 Статус заказа** — обновление на "Оплачено"
- [ ] **1.8 Email** — получение чека на почту

---

## 2. Граничные значения

- [ ] **2.1 Сумма заказа**
  - Минимальная: 0.01
  - Ноль: 0 (ожидается ошибка)
  - Максимальная: значение, которое принимает шлюз
  - Отрицательное: -100 (ожидается ошибка)
- [ ] **2.2 Срок карты**
  - Текущее значение (месяц/год)
  - Более 10 лет
- [ ] **2.3 CVV**
  - 2 цифры
  - 4 цифры

---

## 3. Негативные тесты

- [ ] **3.1 Обязательные поля** — не заполнены
- [ ] **3.2 Номер карты** — неверный (по контрольным цифрам)
- [ ] **3.3 Срок карты** — истекший
- [ ] **3.4 Баланс** — недостаточно средств
- [ ] **3.5 3-D Secure** — пользователь закрыл вкладку
- [ ] **3.6 Чек-бокс** — отсутствие подтверждения
- [ ] **3.7 Имя держателя** — содержит спецсимволы (!"№%;, и т.д.)
- [ ] **3.8 Повторная отправка** — двойной клик по кнопке "Оплатить"

---

## 4. Интеграция с платежным шлюзом

- [ ] **4.1 Успех** — шлюз вернул успех → заказ обновлен, чек пришел
- [ ] **4.2 Ошибка** — шлюз вернул ошибку → заказ не обновлен, чек не пришел
- [ ] **4.3 Неизвестный статус** — шлюз вернул неизвестный статус → обработка ошибки
- [ ] **4.4 Таймаут** — шлюз недоступен → сообщение об ошибке

---
