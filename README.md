# junior-qa-test-task

## Задание 1: Тест-кейсы для формы оплаты заказа

**Файл:** `Тестовое задание Junior QA engineer.xlsx`

### Что покрыто:

| Тип тестов | Количество | ID тест-кейсов |
|---|---|---|
| Позитивные сценарии | 8 | TC-POS-01, TC-01...TC-07 |
| Граничные значения | 4 | TC-BND-01...TC-BND-04 |
| Негативные сценарии | 6 | TC-NEG-01...TC-NEG-06 |
| Интеграция с платежным шлюзом | 3 | TC-INT-01...TC-INT-03 |

### Краткое описание:

- **Позитивные сценарии:** успешная оплата, валидация полей (номер, срок, CVV, имя держателя), работа чек-бокса и ссылки, перенаправление на 3-D Secure, изменение статуса заказа.
- **Граничные значения:** сумма заказа 0, срок карты +10 лет, CVV из 2 и 4 цифр.
- **Негативные сценарии:** пустые поля, неверный номер, истекший срок, недостаточно средств, закрытие окна 3-D Secure, двойной клик.
- **Интеграция:** успешный ответ шлюза, ошибка шлюза, таймаут.

### Структура тест-кейсов:
- ID
- Название
- Приоритет (High/Medium/Low)
- Предусловия
- Шаги воспроизведения
- Ожидаемый результат

## Задание 2: Playwright

**Код теста:** `playwright-test.spec.js`
**Конфигурация:** `playwright.config.js` (запуск в Chromium и Firefox)

**Команды для запуска:**

npx playwright test --project=Chromium
npx playwright test --project=Firefox

**Ожидаемый результат:** Тест успешно проходит в обоих браузерах.
